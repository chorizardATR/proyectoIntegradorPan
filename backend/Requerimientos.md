Requerimientos Funcionales (RF)
•	El sistema debe permitir gestionar clientes: crear, leer, actualizar y desactivar.
•	El sistema debe permitir gestionar propietarios y sus propiedades: crear, leer, actualizar y desactivar.
•	El sistema debe permitir gestionar visitas a propiedades: crear, reasignar, reprogramar, cancelar y registrar resultado.
•	El sistema debe permitir adjuntar y almacenar documentos PDF de exclusividad/contratos por propiedad.
•	El sistema debe permitir administrar imágenes por propiedad, marcando una como portada y ordenándolas.
•	El sistema debe permitir configurar por propiedad los porcentajes de captación y colocación aplicables sobre el precio de cierre.
•	El sistema debe permitir registrar operaciones de cierre (ejemplo: venta) con su tipo, precio de cierre y fecha.
•	El sistema debe calcular y registrar las ganancias del empleado por cada captación o colocación según los porcentajes definidos y el precio de cierre.
•	El sistema debe poder generar reportes de visitas por propiedad en un rango de fechas.
•	El sistema debe poder generar reportes de rendimiento por asesor en función a captaciones, publicaciones, visitas y operaciones cerradas en un rango de fechas.
•	El sistema debe poder generar reportes de ganancias por asesor, por periodo, incluyendo estado de pago (concretado o no) y totales por tipo de operación.
•	El sistema debe permitir que solo el Bróker gestione el CRUD de empleados y la asignación de roles.
•	El sistema debe permitir que el Bróker y la Secretaria gestionen todo lo referente a clientes, propietarios, propiedades, visitas, operaciones, documentos y reportes.
•	El sistema debe publicar el catálogo de propiedades activas con filtros por rango de precio, zona, tipo de operación y estado.
•	El sistema debe mostrar por propiedad su información pública, imágenes y datos relevantes autorizados.
•	El sistema debe ofrecer un botón de contacto por WhatsApp que asigne el asesor de atención usando una cola rotativa entre asesores disponibles.
•	El sistema debe permitir que el asesor inicie sesión y vea su información personal básica y agenda de visitas asignadas con fecha, hora, lugar y estado.
•	El sistema debe permitir que el asesor consulte su rendimiento histórico y del periodo actual basado en captaciones y colocaciones, visitas y operaciones cerradas.
•	El sistema debe permitir que el asesor reciba notificaciones de nuevas visitas o cambios de estado de sus visitas.
•	El sistema debe permitir que el asesor registre el resultado de una visita, es decir una nota.

Requerimientos No Funcionales (RNF)
•	El sistema debe manejar autorización del acceso según su rol: Bróker, Secretaria, Asesor inmobiliario. El cual debe mostrar solo la información correspondiente para cada uno de los usuarios.
•	El sistema debe integrar dos aplicaciones web (administrativa y pública) y una aplicación móvil para asesores.
•	El sistema debe ser modular para separar catálogos, operaciones, agenda/visitas, documentos y reportes.
•	El sistema debe ser responsivo en las interfaces web.
•	El sistema debe usar autenticación con contraseñas cifradas y control de sesión.
•	El sistema debe integrar un modulo móvil para el manejo de la información de los Asesores inmobiliarios.
